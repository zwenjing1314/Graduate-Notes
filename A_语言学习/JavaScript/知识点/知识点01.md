- 最终结果类似于 `"95.3%"`



```js
const resultEl = document.getElementById('result')

resultEl.innerHTML = renderSummary(state.data) + renderPages(state.data);
```

1. resultEl：对象的“遥控器”

* 来源：在代码第 308 行，const resultEl = document.getElementById('result');。
* 含义：它代表了页面上那个 id="result" 的 <div> 元素。你可以把它想象成你手里拿着的一个指向网页上某个区域的“遥控器”。

2. .innerHTML：元素的“内部装修

- 特殊字段：是的，innerHTML 是浏览器提供的 DOM（文档对象模型）接口中一个非常著名的标准属性。
- 作用：它代表该元素开始标签和结束标签之间的所有 HTML 内容。
  - 读操作：如果你执行 console.log(resultEl.innerHTML)，它会告诉你这个 div 里面现在有哪些 HTML 代码。
  - 写操作：当你执行 resultEl.innerHTML = '...' 时，浏览器会做两件事：
    - 清空：把 resultEl 里面原有的所有内容全部删掉。
    - 重绘：把你赋值的字符串当作 HTML 代码解析，并生成新的元素显示在页面上。

为什么要这么写？（数据流向与原理）

- 在这个 OCR 项目中，使用 innerHTML 是实现动态页面更新最直接的方式：
- 初始状态：页面刚加载时，<div id="result"></div> 里面是空的。
- 获取数据：用户提交文件后，JavaScript 从后端拿到了 OCR 结果（JSON 数据）。
- 生成模板：renderSummary 和 renderPages 函数根据这些数据，拼接出一大段包含图片、文字、链接的 HTML 字符串。
- 注入页面：通过 resultEl.innerHTML = ...，这段字符串被瞬间“注入”到页面的 result 区域。

总结对比

写法								含义			形象比喻

resultEl						指向那个 div 盒子本身	这是一个空房间的门牌号

resultEl.innerHTML				指向房间里的家具和装饰	这是房间里的具体摆设

resultEl.innerHTML = '<p>Hi</p>'	把房间清空，换上新家具	重新装修整个房间

## addEventListener() 

addEventListener() 是 JavaScript 中用于监听用户行为或浏览器事件的核心方法。

简单来说，它的作用是让程序“竖起耳朵”等着某个特定的事情发生。一旦这件事发生了，就立即执行你准备好的代码。

1. 语法结构拆解

```js
元素.addEventListener('事件类型', 回调函数);
```

元素 (Element)：你要监听谁？（比如代码中的 thresholdInput 输入框）。
'事件类型' (Event Type)：你要监听什么事？（比如 'input' 表示用户正在打字或拖动滑块）。
回调函数 (Callback Function)：事发生后要做什么？（这里使用了一个箭头函数 () => { ... }）



2. 结合示例代码详细分析

这段代码展示了如何实现**“实时响应”**，我们逐个来看：

A. 监听输入变化 ('input')

```js
thresholdInput.addEventListener('input', () => {
  state.threshold = normalizeThreshold(thresholdInput.value);
  renderResults(); // 重新画图
});
```

原理：每当用户在“置信度阈值”的输入框里改动一个字符，或者拖动滑块时，'input' 事件就会被触发。
数据流向：

- 获取用户最新的输入值。
- 更新内存中的 state 状态。
- 调用 renderResults()，页面瞬间根据新阈值重新筛选并显示单词。

体验：用户不需要点“确定”按钮，看到的是即时的反馈。

B. 监听复选框状态改变 ('change')

```js
onlyLowConfidenceInput.addEventListener('change', () => {
  state.onlyLowConfidence = onlyLowConfidenceInput.checked;
  renderResults();
});
```

- 区别：对于复选框（Checkbox），通常使用 'change' 事件。它在用户勾选或取消勾选的那一瞬间触发。
- .checked：这是复选框特有的属性，返回 true（已勾选）或 false（未勾选）。

## 异步与协程

async/await 并不是让程序“死等”，而是让代码在等待时“暂时让出控制权”，去处理别的事情。

1. ### async 和 await 的区别与关系

可以把它们看作是一对“搭档”，缺一不可：

- async (异步的声明)：
  * 作用：它是一个“标签”，贴在一个函数前面。它告诉 JavaScript 引擎：“这个函数里可能会有一些耗时的操作（比如网络请求），请把它当成一个异步任务来处理。”
  * 副作用：任何被 async 标记的函数，执行后都会自动返回一个 Promise 对象（一种代表未来值的容器）。
- await (等待的关键字)：
  - 作用：它只能用在 async 函数内部。它的意思是：“我要在这里暂停一下，等这个 Promise 有了结果（成功或失败），我再拿着结果继续往下走。”
  - 形象比喻：async 是你走进了一家餐厅（进入了异步环境），await 是你点完餐后坐在座位上等叫号。

2. ### 核心疑问：程序是不是得“等”完才能执行接下来的代码？

这里有一个非常微妙的区别：“当前函数的后续代码” vs “整个程序的其它代码”。

情况 A：对“当前函数”内部而言 —— 是的，它在等
在你的代码中：

```js
const response = await fetch('/ocr'); // 1. 发送请求，然后在这里“暂停”
const data = await response.json();   // 2. 等上面完了，再解析 JSON
statusEl.textContent = '处理完成。';   // 3. 等上面完了，再更新文字
```

在 submit 这个函数内部，代码确实是按顺序执行的。如果没有 await，第 3 行会在第 1 行还没拿到结果时就运行，导致报错。await 保证了逻辑的连贯性。

情况 B：对“整个浏览器/程序”而言 —— 不是，它没在傻等

这是协程（或 JS 的事件循环）最厉害的地方。当代码执行到 await fetch(...) 时：

- 暂停：submit 函数的执行被挂起（Suspended）。
- 让位：JavaScript 引擎会跳出这个函数，去处理其他事情！比如：响应用户的鼠标移动、播放动画、或者执行页面上其他的脚本。
- 回归：等到服务器真的把数据传回来了，JS 引擎会收到一个信号，然后重新回到 await 这一行，把数据赋值给 response，并继续执行第 2、3 行。

结论：它不像传统的“同步阻塞”那样让整个浏览器卡死（白屏），而是一种**“非阻塞式的等待”**。

3. ### 为什么要用这种“协程”写法？

在 async/await 出现之前，我们处理这种等待通常要用“回调地狱”（Callback Hell）：

```js
// 以前的写法（嵌套很深，很难读）
fetch('/ocr').then(response => {
    return response.json();
}).then(data => {
    statusEl.textContent = '处理完成。';
}).catch(error => {
    console.log(error);
});
```

而使用 async/await 后：

- 代码像同步一样直观：你可以从上到下阅读逻辑，就像读 Python 代码一样自然。
- 错误处理更简单：可以直接使用你熟悉的 try...catch 来捕获网络错误，而不需要写一堆 .catch()。

## DOM对象

### 一、什么是 DOM？

**DOM**（Document Object Model，文档对象模型）是浏览器将 HTML / XML 文档解析后生成的**树形结构对象**，供 JavaScript 操作页面内容、结构和样式。

简单说：**HTML 是静态文本，DOM 是内存中的可编程对象树**。

```py
<html>
  <body>
    <div id="box">Hello</div>
  </body>
</html>
```

在内存中会变成：

```
document
  └─ html
      └─ body
          └─ div (id="box")
              └─ #text "Hello"
```

JavaScript 可以通过全局对象 `document` 和 `window` 访问整个 DOM 树。

### 二、DOM 节点类型（Node Types）

DOM 中的每个部分都是“节点（Node）”，常见类型有：

| 元素节点 | 1    | `<div>`、`<p>`                                       |
| -------- | ---- | ---------------------------------------------------- |
| 属性节点 | 2    | `class="box"`（已不常用，现多用 Element.attributes） |
| 文本节点 | 3    | 标签内的文字内容                                     |
| 注释节点 | 8    | `<!-- 注释 -->`                                      |
| 文档节点 | 9    | `document` 本身                                      |
| 文档片段 | 11   | `DocumentFragment`                                   |

**最重要的两种：元素节点（Element）和文本节点（Text）**。

### 三、核心对象：`document` 和 `element`

1. `document` 对象

代表整个 HTML 文档，提供查找、创建元素的方法。

常用查找方法

| 方法                                | 说明               | 返回值                     |
| ----------------------------------- | ------------------ | -------------------------- |
| `getElementById(id)`                | 按 ID 获取单个元素 | Element 或 null            |
| `getElementsByClassName(className)` | 按类名获取元素集合 | HTMLCollection（**动态**） |
| `getElementsByTagName(tag)`         | 按标签名获取       | HTMLCollection             |
| `querySelector(css选择器)`          | 返回第一个匹配元素 | Element 或 null            |
| `querySelectorAll(css选择器)`       | 返回所有匹配元素   | NodeList（**静态**）       |

```js
const box = document.getElementById('box');
const items = document.querySelectorAll('.item');
```

注意：`getElementsBy...` 返回的集合是**实时的**，会随 DOM 变化自动更新；`querySelectorAll` 返回的是**静态快照**。

#### 创建元素的方法

- `createElement(tagName)` —— 创建元素节点
- `createTextNode(text)` —— 创建文本节点
- `createDocumentFragment()` —— 创建文档片段（批量操作时性能更好）

2. `Element` 对象

每个 HTML 标签在 JS 中都是一个 `Element` 实例。

内容操作

| 属性          | 说明                            | 安全性      |
| ------------- | ------------------------------- | ----------- |
| `innerHTML`   | 获取/设置内部 HTML 字符串       | 有 XSS 风险 |
| `textContent` | 获取/设置纯文本（包含隐藏元素） | 安全        |
| `innerText`   | 获取/设置**渲染后**的可见文本   | 会触发重排  |

```js
div.textContent = '<b>安全</b>';  // 会显示 "<b>安全</b>" 字符
div.innerHTML = '<b>加粗</b>';    // 会渲染加粗效果
```
