## Pycharm自动换行

### **编辑器自动换行（适用于代码文件）**‌

- **路径**‌：
   `File` → `Settings` → `Editor` → `General`

- ‌

  操作

  ‌：

  - 勾选 ‌**Use soft wraps in editor**‌
  - （可选）在 ‌**Soft-wrap files**‌ 中添加文件类型，例如：`*.py; *.md; *.txt`
  - （推荐）勾选 ‌**Use original line's indent for wrapped parts**‌ 以保持缩进一致

> 此设置适用于 Python、JSON、Markdown 等多种文件类型

###  **控制台自动换行（适用于运行输出或交互式输入）**‌

- **路径**‌：
   `File` → `Settings` → `Editor` → `General` → `Console`

- ‌

  操作

  ‌：

  - 勾选 ‌**Use soft wraps in console**‌

> 另外，在控制台窗口底部的齿轮图标（⚙️）中也可直接勾选 ‌**Use Soft Wraps**

## Pycharm源代码根目录和测试源代码根目录

在 PyCharm 里，把文件夹标记为 **源代码根目录（Sources Root）** 和 **测试源代码根目录（Test Sources Root）**，核心区别在于：**IDE 会把它们当成不同用途的代码目录来处理**。

### 1. 标记为“源代码根目录”有什么作用

这类目录通常放的是**正式业务代码**，比如：

```py
project/
  src/
    app/
    utils/
```

把 `src` 标记为 `Sources Root` 之后，PyCharm 会：

- **把这个目录加入 import 搜索路径**
   这样你可以直接写：

```py
from app.main import run
```

而不是从更深层或更怪的相对路径导入。

**代码补全、跳转、引用分析更准确**
 PyCharm 会把这里面的模块当成“项目主代码”。

**运行配置时更容易正确识别模块路径**
 特别是使用 `src` 布局时很重要。

**包结构显示更清晰**
 IDE 会按源码目录理解项目结构，而不是把它当普通文件夹。

### 2. 标记为“测试源代码根目录”有什么作用

这类目录通常放的是**测试代码**，比如：

```py
project/
  tests/
    test_user.py
```

把 `tests` 标记为 `Test Sources Root` 后，PyCharm 会：

- **把它识别为测试代码目录**
   测试发现、测试运行、测试导航会更方便。
- **对测试框架支持更友好**
   比如 `pytest`、`unittest` 的识别更自然，右键运行测试更准确。
- **语义上和正式代码分开**
   IDE 知道这里是测试，不是业务代码。
- **某些检查规则会区别对待**
   例如测试代码里未使用的 fixture、测试命名等，IDE 会有更合理的处理。

### 3. 两者共同点

无论是 `Sources Root` 还是 `Test Sources Root`，通常都会影响 **模块解析 / import 路径**。
 也就是说，这两个目录里的代码，PyCharm 都会更认真地参与索引和解析。

但它们的**语义不同**：

- `Sources Root` = 正式源码
- `Test Sources Root` = 测试代码

### 4. 最实际的区别

你可以把它理解成：

- **源代码根目录**：告诉 PyCharm，“这里是我要发布/运行的主代码”
- **测试源代码根目录**：告诉 PyCharm，“这里是专门测试主代码的地方”

这样 PyCharm 在以下方面会更聪明：

- import 解析
- 自动补全
- 跳转定义
- 测试发现与运行
- 项目结构理解

### 5. 一个容易混淆的点

**这两个标记主要影响的是 PyCharm 的项目理解和运行配置，不等于 Python 解释器本身永久改变了搜索路径。**

也就是说：

- 在 PyCharm 里可能正常
- 换到命令行直接运行时，仍然要看你的项目结构、工作目录、`PYTHONPATH`、安装方式等

所以更规范的做法通常还是：

- 使用标准包结构
- 用 `pytest` / `python -m`
- 必要时用可编辑安装 `pip install -e .`
