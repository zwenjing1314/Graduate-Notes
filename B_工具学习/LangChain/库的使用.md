# 介绍

LangChain是一种创新性的框架，是语言模型驱动的应用程序的开发方式。LangChain 是基于大模型的应用开发框架，是一个开源的Python库，旨在通过以下方式更轻松地构建基于LLM的应用程序：

- 向多种不同的基础模型提供通用接口
- 提供管理[Prompt提示](https://zhida.zhihu.com/search?content_id=259375077&content_type=Article&match_order=1&q=Prompt提示&zhida_source=entity)的框架
- 提供长期[记忆能力](https://zhida.zhihu.com/search?content_id=259375077&content_type=Article&match_order=1&q=记忆能力&zhida_source=entity)、外部数据以及其他[代理](https://zhida.zhihu.com/search?content_id=259375077&content_type=Article&match_order=1&q=代理&zhida_source=entity)程序的中央接口，用于处理LLM无法处理的任务（例如计算或搜索）。

通过融合先进原则，LangChain正在重新定义通过传统API可以实现的极限。此外，LangChain应用程序是主动的，使语言模型能够轻松地与其环境交互和适应。Langchain由几个模块组成。正如其名称所示，连接不同的模块在一起是Langchain的主要目的。

# 小记

## .env文件

在开发 LangChain 应用时，`.env`（Environment Variables，环境变量）文件扮演着至关重要的角色。因为 LangChain 几乎总是需要与各种外部大模型提供商（如 OpenAI、Anthropic）或向量数据库（如 Pinecone、Milvus）打交道，而这些服务都需要身份验证。

以下是 `.env` 文件在 LangChain 中的具体作用和工作机理：

### 一、 `.env` 文件的核心作用

1. **保护敏感信息（最重要的一点）**： 在代码中直接写死（Hardcode）API Key（例如 `api_key = "sk-xxx..."`）是非常危险的。如果你不小心把代码传到了 GitHub 等公共仓库，你的 API Key 就会泄露，可能导致你的账户被恶意盗刷。`.env` 文件让你把所有密码、密钥集中存放，并且**绝对不会被提交到代码仓库中**。
2. **简化 LangChain 的初始化**： LangChain 底层做了非常多“约定优于配置”的设计。很多 LangChain 的模块（比如 `ChatOpenAI`）在实例化时，如果在参数中找不到 API Key，就会**自动去系统的环境变量中寻找**对应名称的变量（如 `OPENAI_API_KEY`）。使用 `.env` 文件可以让你的代码非常简洁。
3. **方便环境切换**： 在实际开发中，你可能会有本地开发环境、测试环境和生产环境。通过切换不同的 `.env` 文件，你可以无缝切换不同的数据库地址或测试用的 API 密钥，而不需要修改任何一行业务代码。

### 二、 它的工作机理是什么？

`.env` 文件的本质是一个**普通的纯文本文件**，里面存放着键值对（Key-Value）。它的工作流通常是这样的：

1. **物理存储**：在项目的根目录下创建一个名为 `.env` 的文件，里面写上变量：

```py
OPENAI_API_KEY=sk-xxxxxxxxx
TAVILY_API_KEY=tvly-xxxxxxxxx
```

2. **读取与加载**：你的 Python 程序启动时，通过第三方库（通常是 `python-dotenv`）读取这个文本文件。

3. **注入系统环境**：`python-dotenv` 解析文件内容后，会将这些键值对**临时注入**到当前运行程序的操作系统环境变量字典中（在 Python 中也就是 `os.environ`）。

4. **LangChain 自动提取**：当 LangChain 运行需要调用大模型时，它会通过 `os.environ.get('OPENAI_API_KEY')` 去拉取这个密钥，并带着它向 OpenAI 发起网络请求。

整个过程发生在你程序运行的生命周期内，程序一旦关闭，这些环境变量也就从内存中消失了，不会污染你电脑本身的全局环境变量。

### 三、 实际工作流演示（Python版）

**1. 安装依赖库** 你需要安装读取 `.env` 的库：

```bash
pip install python-dotenv langchain-openai
```

**2. 编写 `.env` 文件** 在项目根目录创建文件，名字就叫 `.env`（没有前缀）：

```py
OPENAI_API_KEY=sk-你的真实OpenAI密钥
```

**3.编写 Python 代码**

```py
import os
from dotenv import load_dotenv
from langchain_openai import ChatOpenAI

# 1. 加载 .env 文件中的变量到系统环境中
load_dotenv()

# 2. 验证一下是否加载成功（仅用于测试，实际开发不需要打出来）
# print(os.getenv("OPENAI_API_KEY")) 

# 3. 实例化大模型
# 注意：我们这里没有传入 api_key 参数！
# ChatOpenAI 底层会自动去环境变量里寻找 "OPENAI_API_KEY"
llm = ChatOpenAI(model="gpt-3.5-turbo")

response = llm.invoke("你好，解释一下你自己。")
print(response.content)
```

### 绝对不能忘记的致命细节：`.gitignore`

为了防止 `.env` 文件被不小心传到网上，你**必须**在你的项目根目录下创建一个 `.gitignore` 文件，并在里面加上一行：

```
.env
```

这样 Git 就会完全忽略这个文件

## 管道操作符 | 的核心概念

### 什么是管道（Pipeline）

管道是一种数据流处理模式，它将多个组件串联起来，前一个组件的输出自动成为后一个组件的输入。
类比现实世界：

```
工厂流水线：
原材料 → [加工站1] → 半成品 → [加工站2] → 成品

数据处理管道：
原始输入 → [Prompt模板] → 格式化消息 → [LLM] → AI回答
```

### 传统写法对比

```py
# ❌ 不使用管道的繁琐方式
step1_messages = prompt.format_messages(input="白小纯是谁？")
step2_response = llm.invoke(step1_messages)
message = step2_response.content

# ✅ 使用管道的简洁方式
chain = prompt | llm
message = chain.invoke({"input": "白小纯是谁？"}).content
```

### 管道操作的实现原理

1. #### 底层机制：运算符重载

Python 允许类通过定义特殊方法来自定义运算符行为。LangChain 利用了这一点：

```py
# LangChain 内部实现（简化版）
class Runnable:
    def __or__(self, other):
        """当使用 self | other 时调用此方法"""
        return RunnableSequence(self, other)
```

数据流向：

```
执行: prompt | llm
     ↓
触发: prompt.__or__(llm)
     ↓
返回: RunnableSequence(prompt, llm) 对象
     ↓
这个对象保存了两个组件的引用
```

2. RunnableSequence 类的核心结构

```py
class RunnableSequence(Runnable):
    def __init__(self, *steps):
        self.steps = list(steps)
    
    def invoke(self, input, config=None):
        """顺序执行所有步骤"""
        intermediate = input
        
        for i, step in enumerate(self.steps):
            try:
                # 执行当前步骤
                intermediate = step.invoke(intermediate, config)
            except Exception as e:
                # 添加上下文信息
                raise RuntimeError(
                    f"Error in step {i}: {step.__class__.__name__}"
                ) from e
        
        return intermediate
    
    def __or__(self, other):
        """支持继续链接"""
        return RunnableSequence(*self.steps, other)
```

### 示例

```py
from langchain_core.output_parsers import JsonOutputParser
from langchain_core.prompts import ChatPromptTemplate
from langchain_openai import ChatOpenAI

# 四段式管道
prompt = ChatPromptTemplate.from_messages([
    ("system", "你是一个JSON生成器"),
    ("user", "生成关于{topic}的信息")
])

llm = ChatOpenAI(model="glm-4", api_key=api_key, base_url=base_url)
json_parser = JsonOutputParser()

# 组合管道
chain = prompt | llm | json_parser

result = chain.invoke({"topic": "Python编程语言"})
# 返回: {"name": "Python", "creator": "Guido van Rossum", ...}
```

## invoke() 方法

### invoke() 的核心定义

invoke() 是 LangChain 中 Runnable 接口的核心方法，它的作用是：
	接收输入数据，执行组件的处理逻辑，返回处理结果
这是一个同步的、单次执行的方法。

不同组件中 invoke() 的具体行为

#### ChatPromptTemplate.invoke() - 提示词模板

```py
prompt = ChatPromptTemplate.from_messages([
    ("system", "你是一个AI助手"),
    ("user", "{input}")
])

result = prompt.invoke({"input": "你好"})
```

数据流向详解：

```py
第1步：接收输入
输入: {"input": "你好"}
     ↓
第2步：变量替换
遍历模板中的每条消息：
  
  消息1: SystemMessage("你是一个AI助手")
    ├─ 检查是否有变量占位符: 无
    └─ 保持不变
  
  消息2: HumanMessage("{input}")
    ├─ 检测到变量: {input}
    ├─ 从输入字典中查找 "input" 键
    ├─ 替换: "{input}" → "你好"
    └─ 生成: HumanMessage("你好")
     ↓
第3步：构建消息列表
[
    SystemMessage(content="你是一个AI助手"),
    HumanMessage(content="你好")
]
     ↓
第4步：返回结果
返回类型: List[BaseMessage]
```

关键特点：

- 输入：字典（包含变量值）
- 输出：消息列表（已填充变量）
- 不涉及网络请求，纯本地操作

#### ChatOpenAI.invoke() - 大语言模型

```py
llm = ChatOpenAI(model="glm-4", api_key=api_key, base_url=base_url)

# 方式1：直接传入字符串
result = llm.invoke("你好")

# 方式2：传入消息列表
from langchain_core.messages import HumanMessage
result = llm.invoke([HumanMessage(content="你好")])
```

数据流向详解：

```py
第1步：接收输入
输入可以是：
  - 字符串: "你好"
  - 消息列表: [HumanMessage("你好")]
     ↓
第2步：输入标准化
如果输入是字符串：
  └─ 转换为: [HumanMessage(content="你好")]
  
如果输入已是消息列表：
  └─ 保持不变
     ↓
第3步：构建 HTTP 请求
构造请求体和头部：

URL: https://open.bigmodel.cn/api/paas/v4/chat/completions
Headers: {
    "Authorization": "Bearer <api_key>",
    "Content-Type": "application/json"
}
Body: {
    "model": "glm-4",
    "messages": [
        {"role": "user", "content": "你好"}
    ],
    "temperature": 0.7,
    "max_tokens": 1024
}
     ↓
第4步：发送网络请求
HTTPS POST 请求 → 智谱云服务器
     ↓
第5步：等待响应
阻塞等待服务器返回（受 timeout 参数限制）
     ↓
第6步：接收响应
HTTP 200 OK
Response Body:
{
    "id": "chatcmpl-xxx",
    "choices": [{
        "index": 0,
        "message": {
            "role": "assistant",
            "content": "你好！有什么可以帮助你的？"
        },
        "finish_reason": "stop"
    }],
    "usage": {
        "prompt_tokens": 5,
        "completion_tokens": 10,
        "total_tokens": 15
    }
}
     ↓
第7步：解析响应
提取 choices[0].message.content
创建 AIMessage 对象：
AIMessage(
    content="你好！有什么可以帮助你的？",
    response_metadata={
        "token_usage": {...},
        "model_name": "glm-4",
        ...
    }
)
     ↓
第8步：返回结果
返回类型: AIMessage
```

关键特点：

- 输入：字符串或消息列表
- 输出：AIMessage 对象
- 涉及网络请求，会阻塞等待
- 可能抛出网络异常、API 错误等

#### StrOutputParser.invoke() - 输出解析器

```py
from langchain_core.output_parsers import StrOutputParser

parser = StrOutputParser()

# 通常接收 LLM 的输出
from langchain_core.messages import AIMessage
result = parser.invoke(AIMessage(content="这是回答"))
```

数据流向详解：

```
第1步：接收输入
输入: AIMessage(content="这是回答")
     ↓
第2步：提取内容
访问 .content 属性
     ↓
第3步：返回纯字符串
返回: "这是回答"
返回类型: str
第1步：接收输入
输入: AIMessage(content="这是回答")
     ↓
第2步：提取内容
访问 .content 属性
     ↓
第3步：返回纯字符串
返回: "这是回答"
返回类型: str
```

关键特点：

- 输入：AIMessage 或其他消息对象
- 输出：纯字符串
- 简单的数据转换，无网络请求

# langchain_openai

## ChatOpenAI

一、ChatOpenAI 的核心作用

ChatOpenAI 是 LangChain 提供的一个聊天模型封装类，它的作用是：
统一接口抽象：无论底层是 OpenAI、智谱、还是其他兼容服务，都用相同的 API
管理HTTP连接：处理认证、重试、超时等网络细节
标准化输入输出：将消息转换为标准格式，将响应解析为统一对象

二、初始化过程详解

示例

```py
llm = ChatOpenAI(
    model="glm-4",
    api_key=api_key,
    base_url=base_url,
    timeout=20,
    max_retries=1,
)
```

1. model="glm-4" - 指定模型版本
作用：告诉API服务器使用哪个模型进行推理
数据流向：

```
  用户代码传入 "glm-4"
       ↓
  存储在 ChatOpenAI 实例的 self.model_name 属性
       ↓
  调用时放入 HTTP 请求体: {"model": "glm-4", ...}
       ↓
  智谱服务器根据此字段路由到 GLM-4 模型服务
```

2. api_key=api_key - 身份认证凭证
作用：证明你有权限调用API（从环境变量读取）
操作原理：

```py
  # 内部会将 api_key 转换为 HTTP 请求头
  headers = {
      "Authorization": f"Bearer {api_key}",
      "Content-Type": "application/json"
  }
```

安全考虑：不应硬编码在代码中，而是通过 .env 文件管理

3. base_url=base_url - API端点地址
作用：指定请求发送到哪个服务器
当前值："https://open.bigmodel.cn/api/paas/v4"（智谱的API地址）
对比：
	OpenAI官方："https://api.openai.com/v1"
	智谱："https://open.bigmodel.cn/api/paas/v4"
	本地部署："http://localhost:8000/v1"

4. timeout=20 - 请求超时时间
作用：如果20秒内服务器未响应，则抛出异常
单位：秒
数据流向：

```py
  发起HTTP请求 → 开始计时 → 等待响应
       ↓ (20秒内收到响应)          ↓ (超过20秒)
  正常返回结果                抛出 TimeoutError
```

为什么需要：防止网络问题导致程序无限等待

5. max_retries=1 - 最大重试次数
作用：请求失败时自动重试的次数
场景：
	网络抖动导致连接中断
	服务器临时返回503错误
工作流程：

```
  第一次请求 → 失败
       ↓
  等待短暂时间（指数退避）
       ↓
  第二次请求（重试1次）→ 成功/继续失败
       ↓ (如果还失败且 max_retries=1)
  抛出异常
```

```
│ 第9步：提取内容                              │
│ .content 属性获取纯文本                       │
│ message = "白小纯是..."               │
└─────────────────────────────────────────────┘
```

# langchain_core

## prompts

### ChatPromptTemplate

ChatPromptTemplate 是 LangChain 提供的聊天提示词模板类，它的主要作用是：

功能1：结构化消息管理
将不同角色的消息（system、user、assistant）组织成标准格式

功能2：变量填充
支持用 {变量名} 占位，运行时动态替换

功能3：格式转换
自动将简单元组转换为标准的 Message 对象

功能4：链式组合
可以与 LLM、OutputParser 等组件通过 | 操作符连接

#### from_messages()

方法签名

```py
ChatPromptTemplate.from_messages(messages: List) -> ChatPromptTemplate
```

输入参数解析
from_messages() 接收一个消息列表，每个消息可以是以下格式之一：

格式1：元组形式（最常用）

```
("角色", "消息内容")
```

示例：

```py
prompt = ChatPromptTemplate.from_messages([
    ("system", "你是一个AI助手，请简短回答用户问题"),
    ("user", "{input}")
])
```

数据流向与内部处理：

```
第1步：接收元组列表
[
    ("system", "你是一个AI助手，请简短回答用户问题"),
    ("user", "{input}")
]
     ↓
第2步：遍历每个元组，根据角色创建对应的 Message 对象
     ↓
┌─────────────────────────────────────────────┐
│ ("system", "...") → SystemMessage 对象      │
│   - role: "system"                          │
│   - content: "你是一个AI助手..."             │
│   - 作用：设定AI的行为准则和角色定位          │
└─────────────────────────────────────────────┘
     ↓
┌─────────────────────────────────────────────┐
│ ("user", "{input}") → HumanMessage 对象     │
│   - role: "user"                            │
│   - content: "{input}" （包含占位符）        │
│   - 作用：表示用户的输入，等待运行时填充      │
└─────────────────────────────────────────────┘
     ↓
第3步：将所有 Message 对象存入模板
     ↓
返回 ChatPromptTemplate 实例
  - self.messages = [SystemMessage(...), HumanMessage(...)]
  - self.input_variables = ["input"]  # 自动检测到的变量列表
```

格式2：字典形式

```py
prompt = ChatPromptTemplate.from_messages([
    {"role": "system", "content": "你是一个AI助手"},
    {"role": "user", "content": "{input}"}
])
```

格式3：直接使用 Message 对象

```py
from langchain_core.messages import SystemMessage, HumanMessage

prompt = ChatPromptTemplate.from_messages([
    SystemMessage(content="你是一个AI助手"),
    HumanMessage(content="{input}")
])
```

## output_parsers

### JsonOutputParser()

 `JsonOutputParser()` 是 LangChain 提供的输出解析器，用来解析大模型返回的 JSON 格式内容。

在链式调用中：

  ```python
  chain = prompt | llm | output_parser
  ```

  执行流程是：

  1. prompt：把用户输入整理成提示词
  2. llm：调用大模型生成回答
  3. output_parser：把模型返回的文本解析成 Python 对象



```
  例如模型返回：

  {
    "question": "LangChain是什么?",
    "answer": "LangChain 是一个用于构建大语言模型应用的框架。"
  }

  经过 JsonOutputParser() 解析后，得到的是 Python 字典：

  {
      "question": "LangChain是什么?",
      "answer": "LangChain 是一个用于构建大语言模型应用的框架。"
  }

  这样后续就可以直接用字典方式取值：

  message["question"]
  message["answer"]
```

 #### 为什么需要提示模型返回 JSON

  JsonOutputParser() 只负责“解析 JSON”，不会自动把普通文本变成 JSON。

  如果模型返回的是普通自然语言：

  LangChain 是一个用于构建大语言模型应用的框架。

  这不是合法 JSON，JsonOutputParser() 就会报错。

  所以使用 JsonOutputParser() 时，通常要在提示词里明确要求模型返回 JSON，例如：

```py
  message = chain.invoke({
      "input": "LangChain是什么? 用JSON格式回复，问题用question，回答用answer"
  })
```

  #### 和 StrOutputParser 的区别

  StrOutputParser() 会把模型输出当成普通字符串处理。

  JsonOutputParser() 会把模型输出当成 JSON 处理，并转换成 Python 对象。

  简单理解：

```py
  StrOutputParser()
  # 输出: "LangChain 是一个用于构建大语言模型应用的框架。"

  JsonOutputParser()
  # 输出: {"question": "...", "answer": "..."}
```



  #### 总结

  JsonOutputParser() 的核心作用是：

  把模型输出的 JSON 字符串解析成 Python 数据结构，方便程序继续处理。

  使用它时要保证模型真的返回合法 JSON，否则会解析失败并报错。
