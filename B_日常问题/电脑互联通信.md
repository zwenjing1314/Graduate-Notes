## 用局域网共享（像同一台网盘一样）

Win 做“共享盘”

适合：经常互传文件、希望在“文件管理器/访达里直接拖拽”。

### A. Windows 开 SMB 共享（推荐：Win 做“共享盘”）

**Windows 11：**

1. 选一个文件夹 → 右键 **属性** → **共享** → **高级共享** → 勾选“共享此文件夹”
2. **权限**里给你的账户读写（或临时给 Everyone 读写，传完再收回）
3. 打开 **设置 → 网络和 Internet → 高级网络设置 → 高级共享设置**
   - 开启 **网络发现**
   - 开启 **文件和打印机共享**
4. 记下 Windows 的 IP（命令行 `ipconfig`，看 IPv4）

**Mac 访问：**

- 打开 **访达 → 前往 → 连接服务器…**
- 输入：`smb://<Windows的IP>/<共享名>`（如：smb://192.168.1.23/Share）
- 用 Windows 用户名/密码登录后，就像挂载了一个磁盘，直接拖文件。

### B. Mac 开 SMB 共享（Mac 做“共享盘”）

**Mac：**

1. **系统设置 → 通用 → 共享 → 文件共享** 打开
2. 点 **i** 或“选项”，勾选 **使用 SMB 共享文件和文件夹**
3. 添加要共享的文件夹，设置用户权限

**Windows 访问：**

- 文件资源管理器地址栏输入：`\\<Mac的IP>\`
- 或 **映射网络驱动器**，以后像本地盘一样用。

✅ 优点：最快、最像“同一个文件夹”；大文件也稳。
 ⚠️ 注意：两台要在同一个局域网（同一个路由器/同一 Wi-Fi），并允许防火墙放行 SMB。

### C. Linux 端 配置Samba共享

**安装Samba服务**

```bash
sudo apt-get update
sudo apt-get install samba samba-commo
sudo apt install samba --reinstall  # 安装主服务包
```

**创建共享文件夹**

```bash
sudo mkdir /home/yourname/shared
sudo chmod 777 /home/yourname/shared  # 设置权限为可读
```

**编辑Samba配置文件**

```bash
sudo nano /etc/samba/smb.conf
```

在文件末尾添加以下配置：

```bash
[shared]
path = /home/yourname/shared
valid users = yourname
writable = yes
browsable = yes
guest ok = no
```

**创建Samba用户并设置密码**

```bash
sudo smbpasswd -a yourname
```

输入并确认Samba密码（可以与Linux登录密码不同）

**重启Samba服务**

```bash
sudo systemctl restart smbd
sudo systemctl restart nmbd
```

**配置防火墙（如果启用）**

```bash
sudo ufw allow 139/tcp
sudo ufw allow 445/tcp
sudo ufw reload
```

**获取Linux的IP地址**

```py
ip addr show
```

记录IPv4地址（如192.168.1.100）

**在Mac端连接共享文件夹**

1. **打开Finder**
2. **点击顶部菜单栏的"前往" → "连接服务器"**
3. **输入连接地址**

```bash
smb://192.168.1.100/shared
```

（将IP地址替换为你的Linux实际IP）

**输入凭据**

- 用户名：yourname（你在Linux上创建的用户名）
- 密码：你为Samba设置的密码

**连接成功后**

- 共享文件夹将显示在Finder中
- 你可以像操作本地文件一样进行读写操作



smb.conf文件出现了一点问题

先备份旧文件：

```bash
sudo cp /etc/samba/smb.conf /etc/samba/smb.conf.bak.$(date +%F-%H%M%S)
```

然后把 `/etc/samba/smb.conf` 改成下面这份“干净的最小版”：

```
[global]
   workgroup = WORKGROUP
   server string = %h server (Samba, Ubuntu)
   server role = standalone server
   map to guest = never

[shared]
   path = /home/wenjing/shared
   browseable = yes
   read only = no
   valid users = wenjing
   force user = wenjing
```

注意两点：

- **不要再写行尾 `# 注释`**。要注释就单独另起一行。
- 这份配置走的是**用户名密码访问**，比访客共享稳一些。

接着确保目录和权限没问题：

```bash
mkdir -p /home/wenjing/shared
chmod 755 /home/wenjing/shared
```

然后给 Samba 加密码。Ubuntu/Samba 的共享访问通常还需要把系统用户加入 Samba 用户库。

```bash
sudo smbpasswd -a wenjing
```

再用 `testparm` 检查配置语法。Ubuntu/Samba 文档和 man page 都建议改完先跑这个检查。

```bash
testparm
```

如果你机器上提示 `testparm: command not found`，装一下：

```bash
sudo apt update
sudo apt install samba samba-common-bin
```

最后重启 Samba 服务。Ubuntu 官方文档给出的命令是：

```bash
sudo systemctl restart smbd.service nmbd.service
```

然后在 Mac 上连：

```
smb://你的Ubuntu局域网IP/shared
```
