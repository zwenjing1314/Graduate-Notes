@torch.inference_mode() 装饰器

作用： 这是 PyTorch 提供的装饰器，用于禁用梯度计算和版本计数器，专门用于推理场景。
与 @torch.no_grad() 的区别：

- @torch.no_grad(): 只禁用梯度计算
- @torch.inference_mode(): 禁用梯度计算 + 禁用版本计数器（更严格，性能更好）

在代码中的作用：

```py
@torch.inference_mode()
def encode_images(self, image_paths: Sequence[Path], batch_size: int = 32) -> np.ndarray:
    # ... 推理代码 ...
```

这个装饰器确保：

1. 不计算梯度：节省内存，加速计算
2. 禁用版本计数器：某些操作会检查张量是否被修改过，推理时不需要这个检查
3. 防止意外训练：即使模型中有可训练参数，也不会更新

用法示例：

```py
import torch
import torch.nn as nn

model = nn.Linear(10, 2)

# 方式 1：作为装饰器
@torch.inference_mode()
def predict(x):
    return model(x)

# 方式 2：作为上下文管理器
def predict2(x):
    with torch.inference_mode():
        return model(x)

# 对比 no_grad
@torch.no_grad()
def predict3(x):
    return model(x)
```

## 常见API

### squeeze()

squeeze() 用于压缩维度，即移除张量中维度为 1 的维度。

两种用法：
squeeze() - 移除所有维度为 1 的维度
squeeze(dim) - 移除指定维度中维度为 1 的维度（如果该维度不为 1，则不执行任何操作）

具体示例：

```py
import torch
import numpy as np

# PyTorch 中的 squeeze
x = torch.randn(1, 3, 1, 4)
print(f"原始形状：{x.shape}")  # torch.Size([1, 3, 1, 4])

y = x.squeeze(0)  # 移除第 0 维（维度为 1）
print(f"squeeze(0) 后：{y.shape}")  # torch.Size([3, 1, 4])

z = x.squeeze()  # 移除所有维度为 1 的维度
print(f"squeeze() 后：{z.shape}")  # torch.Size([3, 4])

# NumPy 中也有类似的 squeeze 函数
x_np = np.random.randn(1, 3, 1, 4)
print(f"\nNumPy 原始形状：{x_np.shape}")  # (1, 3, 1, 4)

y_np = np.squeeze(x_np, axis=0)
print(f"np.squeeze(axis=0) 后：{y_np.shape}")  # (3, 1, 4)

z_np = np.squeeze(x_np)
print(f"np.squeeze() 后：{z_np.shape}")  # (3, 4)
```

### view()

基本语法

```
tensor.view(shape)  # 或 view(dim1, dim2, ...)
```

核心规则：

- 不改变数据，只改变张量的"视角"（类似 NumPy 的 reshape）
- 总元素数必须保持不变
- -1 表示"自动计算该维度"

常见用法示例：

```py
import torch

x = torch.randn(4, 16, 5, 5)  # 6400 个元素

# 用法 1: 展平成 2D
y = x.view(-1, 16 * 5 * 5)    # (4, 400)
y = x.view(4, -1)             # (4, 400) 等价

# 用法 2: 改变维度顺序（需配合 permute）
x = torch.randn(2, 3, 4)
y = x.view(2, 12)             # (2, 24) ✓
z = x.view(6, 4)              # (6, 4) ✓

# 用法 3: 批量操作
batch_size = 32
features = x.view(batch_size, -1)  # 保持 batch，展平其他
```

### max()

是 PyTorch 中用于获取张量最大值及其索引的函数。让我详细解释这一步操作的含义。

```py
_, preds = torch.max(outputs, 1)
```

这行代码的作用是：
torch.max(outputs, 1) 沿着维度1（即特征维度）寻找最大值
返回两个值：最大值本身和对应的索引
使用 _ 忽略最大值，只保留索引（预测的类别）
preds 变量存储了模型对每个输入样本的预测类别
在分类任务中，outputs 通常是模型最后一层的输出，形状为 [batch_size, num_classes]，其中每一行表示一个样本在各个类别上的预测分数。通过 torch.max(outputs, 1)，我们找到每个样本预测分数最高的类别作为最终预测结果。

例如，如果 outputs 是：

```
[[0.1, 0.8, 0.1],  # 样本0: 类别1得分最高
 [0.7, 0.2, 0.1],  # 样本1: 类别0得分最高
 [0.2, 0.3, 0.5]]  # 样本2: 类别2得分最高
```

那么 torch.max(outputs, 1) 将返回：
最大值: [0.8, 0.7, 0.5]
索引(preds): [1, 0, 2]
所以 preds 就是模型对每个样本的预测类别。

### sum()

是 PyTorch 中用于计算张量元素总和的函数。让我详细解释这行代码的具体用法和工作原理。

```py
running_corrects += torch.sum(preds == labels.data)
```

running_corrects += 将当前批次的正确预测数累加到总的正确数中，用于计算整个 epoch 的准确率。

完整示例
假设有一个批次大小为 4 的数据：

```py
# 模型输出（每个类别的得分）
outputs = tensor([[0.8, 0.2],    # 样本 0: 预测类别 0
                  [0.3, 0.7],    # 样本 1: 预测类别 1
                  [0.6, 0.4],    # 样本 2: 预测类别 0
                  [0.1, 0.9]])   # 样本 3: 预测类别 1

# 获取预测类别
_, preds = torch.max(outputs, 1)  # preds = [0, 1, 0, 1]

# 真实标签
labels = [0, 1, 1, 1]

# 比较预测和真实标签
preds == labels  # 结果：[True, True, False, True]

# 统计正确数量
torch.sum(preds == labels)  # 结果：tensor(3)
```

### scheduler.step() 

 是 PyTorch 中学习率调度器的核心方法，用于动态调整优化器的学习率。让我详细解释它的用法和工作原理。

基本作用
scheduler.step() 的主要功能是：
每个 epoch 结束后自动调整学习率
根据预定义的调度策略逐渐降低（或改变）学习率
帮助模型在训练后期更精细地收敛到最优解

工作流程

```py
# 初始化时设置初始学习率
optimizer = optim.SGD(model.parameters(), lr=0.001)

# 定义调度器（以 StepLR 为例）
scheduler = lr_scheduler.StepLR(optimizer, step_size=7, gamma=0.1)

# 每个 epoch 结束时调用
for epoch in range(num_epochs):
    # ... 训练代码 ...
    
    if phase == 'train':
        scheduler.step()  # 调整学习率
```
