# pdf 单位换算

1. PDF 的坐标系
PDF 使用 Point（点） 作为基本单位
1 英寸 = 72 Points（这是 PDF 的标准定义）
PDF 页面尺寸通常表示为：A4 = 595 × 842 points

2. DPI 的含义
DPI (Dots Per Inch)：每英寸的像素数
dpi=300 表示：1 英寸 = 300 像素

3. 计算公式推导

```py
zoom = dpi / 72.0
```

数学推导过程：

```text
已知：
- 1 inch = 72 points
- 1 inch = 300 pixels (当 dpi=300)

求：1 point = ? pixels

解：
1 point = (1/72) inch
        = (1/72) × 300 pixels
        = 300/72 pixels
        ≈ 4.167 pixels

所以：zoom = 300/72 ≈ 4.167
```

实际应用场景

假设 PDF 页面尺寸为 A4（595 × 842 points）：

```py
# 原始尺寸（points）
pdf_width = 595 points
pdf_height = 842 points

# 应用 zoom = 300/72 ≈ 4.167
output_width = 595 × 4.167 ≈ 2479 pixels
output_height = 842 × 4.167 ≈ 3508 pixels

# 验证 DPI
2479 pixels / (595/72 inches) ≈ 300 DPI ✓
```

为什么需要这个转换？

```py
PDF (矢量)                    渲染后的图片 (位图)
┌──────────────┐             ┌─────────────────┐
│              │             │                 │
│  595 pts     │  zoom=4.17  │  2479 pixels    │
│  ×           │ ──────────→ │  ×              │
│  842 pts     │             │  3508 pixels    │
│              │             │                 │
└──────────────┘             └─────────────────┘
   72 pts/inch                  300 pixels/inch
```

关键原因：
OCR 需要高分辨率：低分辨率会导致文字识别率下降
保持物理尺寸一致：通过 zoom 确保渲染后的图片在相同物理尺寸下有足够像素
坐标映射：后续 OCR 返回的 bbox 坐标需要能准确映射回原图

#  Otsu 算法

 Otsu（大津）算法自动计算图像二值化的最佳阈值

```py
def _otsu_threshold(image: Image.Image) -> int:
    """用 Otsu（大津）算法自动计算图像二值化的最佳阈值。
    
    核心思想：
    --------
    Otsu 算法通过遍历所有可能的灰度阈值（0-255），找到一个最优的分割点，
    使得分割后的"背景"和"前景"两类像素之间的差异最大化。
    
    数学原理：
    --------
    - 将像素按阈值分为两类：背景（灰度 <= 阈值）和前景（灰度 > 阈值）
    - 计算两类的类间方差：σ²_B = w₀ × w₁ × (μ₀ - μ₁)²
      其中：
      - w₀, w₁：背景和前景的像素权重（占比）
      - μ₀, μ₁：背景和前景的平均灰度值
    - 选择使类间方差最大的阈值作为最佳分割点
    
    应用场景：
    --------
    在 OCR 预处理中，将灰度图像转换为纯黑白二值图像，
    提高文字识别准确率，特别适用于扫描件、传真等低质量文档。
    
    参数：
    ----
    image : PIL.Image.Image
        输入的灰度图像（单通道，像素值范围 0-255）
        
    返回：
    ----
    int
        最佳阈值（0-255 之间的整数）
        - 灰度值 < 阈值的像素将被视为"背景"（转为黑色 0）
        - 灰度值 >= 阈值的像素将被视为"前景"（转为白色 255）
    
    示例：
    ----
    >>> from PIL import Image
    >>> img = Image.open("document.jpg").convert("L")
    >>> threshold = _otsu_threshold(img)
    >>> print(f"最佳阈值: {threshold}")  # 例如: 128
    >>> binary_img = img.point(lambda p: 255 if p > threshold else 0)
    """
    
    # ==================== 步骤 1: 获取图像的灰度直方图 ====================
    histogram = image.histogram()
    # histogram 是一个长度为 256 的列表
    # 索引 i (0-255) 表示灰度值
    # 值 histogram[i] 表示灰度值为 i 的像素数量
    # 
    # 例如：histogram[0] = 500  → 有 500 个像素是纯黑色（灰度 0）
    #       histogram[255] = 800 → 有 800 个像素是纯白色（灰度 255）
    
    # ==================== 步骤 2: 计算总像素数 ====================
    total = sum(histogram)
    # total = 所有灰度级的像素数之和 = 图像的总像素数
    # 例如：一张 100×100 的图片，total = 10000
    
    # 边界情况处理：如果图像为空或无效，返回默认阈值 180
    if total <= 0:
        return 180  # 防御性编程：避免除以零错误
    
    # ==================== 步骤 3: 计算全局灰度总和 ====================
    total_sum = sum(value * count for value, count in enumerate(histogram))
    # total_sum = Σ(灰度值 × 该灰度的像素数)
    #          = 0×histogram[0] + 1×histogram[1] + ... + 255×histogram[255]
    # 
    # 物理意义：图像中所有像素的灰度值之和
    # 用途：后续用于快速计算前景的平均灰度（通过减法而非重新累加）
    # 
    # 例如：如果有 100 个像素灰度为 50，贡献为 50×100 = 5000
    
    # ==================== 步骤 4: 初始化变量 ====================
    background_sum = 0.0      # 背景部分的灰度总和（累加器）
    background_weight = 0     # 背景部分的像素数量（累加器）
    best_threshold = 180      # 记录最佳阈值（初始化为默认值）
    best_variance = -1.0      # 记录最大类间方差（初始化为负数，确保第一次比较会更新）
    
    # ==================== 步骤 5: 遍历所有可能的阈值（0-255）====================
    for value, count in enumerate(histogram):
        # value: 当前候选阈值（也是当前灰度级，0-255）
        # count: 灰度值为 value 的像素数量
        
        # ---------- 5.1: 累加背景统计信息 ----------
        background_weight += count
        # background_weight = 灰度值 <= value 的所有像素数量
        # 随着 value 从 0 增加到 255，background_weight 逐渐增大
        # 
        # 例如：
        # value=0 时: background_weight = histogram[0]
        # value=1 时: background_weight = histogram[0] + histogram[1]
        # value=2 时: background_weight = histogram[0] + histogram[1] + histogram[2]
        
        # 如果当前灰度级没有像素，跳过（避免除以零）
        if background_weight == 0:
            continue
        
        # ---------- 5.2: 计算前景统计信息 ----------
        foreground_weight = total - background_weight
        # foreground_weight = 灰度值 > value 的像素数量
        # 
        # 逻辑：总像素数 - 背景像素数 = 前景像素数
        # 
        # 例如：total=10000, background_weight=3000
        #       → foreground_weight = 7000
        
        # 如果前景没有像素了，说明阈值已经太大，停止遍历
        if foreground_weight == 0:
            break  # 所有像素都被分到背景中，继续遍历没有意义
        
        # ---------- 5.3: 更新背景灰度总和 ----------
        background_sum += value * count
        # background_sum = 背景部分（灰度 <= value）所有像素的灰度值之和
        # 
        # 例如：
        # value=0 时: background_sum = 0 × histogram[0]
        # value=1 时: background_sum = 0×histogram[0] + 1×histogram[1]
        # value=2 时: background_sum = 0×h[0] + 1×h[1] + 2×h[2]
        
        # ---------- 5.4: 计算背景和前景的平均灰度 ----------
        background_mean = background_sum / background_weight
        # background_mean (μ₀) = 背景部分的平均灰度值
        # 
        # 例如：background_sum=150000, background_weight=3000
        #       → background_mean = 50.0（背景偏暗）
        
        foreground_mean = (total_sum - background_sum) / foreground_weight
        # foreground_mean (μ₁) = 前景部分的平均灰度值
        # 
        # 技巧：利用 total_sum - background_sum 快速得到前景灰度总和
        # 而不是重新遍历前景像素累加
        # 
        # 例如：total_sum=2000000, background_sum=150000
        #       → foreground_sum = 1850000
        #       → foreground_mean = 1850000 / 7000 ≈ 264.3（前景偏亮）
        
        # ---------- 5.5: 计算类间方差（Between-class Variance）----------
        between_variance = (
                background_weight
                * foreground_weight
                * (background_mean - foreground_mean)
                * (background_mean - foreground_mean)
        )
        # 公式：σ²_B = w₀ × w₁ × (μ₀ - μ₁)²
        # 
        # 各项含义：
        # - w₀ (background_weight):     背景像素数量（权重）
        # - w₁ (foreground_weight):     前景像素数量（权重）
        # - μ₀ (background_mean):       背景平均灰度
        # - μ₁ (foreground_mean):       前景平均灰度
        # - (μ₀ - μ₁)²:                 两类均值差的平方
        # 
        # 物理意义：
        # - 方差越大 → 背景和前景差异越明显 → 分割效果越好
        # - 方差越小 → 背景和前景混杂 → 分割效果差
        # 
        # 直观理解：
        # 1. 如果背景和前景都很"纯净"（均值差异大），方差大 ✓
        # 2. 如果背景和前景都很"混杂"（均值接近），方差小 ✗
        # 3. 如果某一类像素很少（权重小），方差也会小（惩罚不平衡分割）
        
        # ---------- 5.6: 更新最佳阈值 ----------
        if between_variance > best_variance:
            # 如果当前阈值的类间方差比之前记录的最大值还大
            best_variance = between_variance  # 更新最大方差
            best_threshold = value            # 更新最佳阈值
            # 
            # 这保证了循环结束后，best_threshold 是使类间方差最大的那个阈值
    
    # ==================== 步骤 6: 返回最佳阈值 ====================
    return best_threshold
    # 例如：返回 128
    # 意味着：
    # - 灰度值 < 128 的像素 → 判定为"背景" → 二值化后变为黑色 (0)
    # - 灰度值 >= 128 的像素 → 判定为"前景" → 二值化后变为白色 (255)
    # 
    # 这个阈值会被上层函数用于图像二值化：
    # grayscale.point(lambda pixel: 255 if pixel > threshold else 0)
```

工作原理（数据流）

```py
输入：灰度图像（像素值 0-255）
       ↓
┌──────────────────────────────────┐
│ 步骤 1: 统计直方图                │
│ histogram[0..255]                │
│ 记录每个灰度级的像素数量          │
└──────────────┬───────────────────┘
               ↓
┌──────────────────────────────────┐
│ 步骤 2: 遍历所有可能的阈值        │
│ for threshold in range(256):     │
│                                  │
│   将像素分为两类：                │
│   - 背景: 灰度 <= threshold      │
│   - 前景: 灰度 > threshold       │
│                                  │
│   计算类间方差：                  │
│   σ²_B = w₀ × w₁ × (μ₀-μ₁)²   │
│                                  │
│   如果方差更大：                  │
│     更新最佳阈值                  │
└──────────────┬───────────────────┘
               ↓
┌──────────────────────────────────┐
│ 步骤 3: 返回最佳阈值              │
│ 使背景和前景差异最大的那个值      │
└──────────────┬───────────────────┘
               ↓
输出：最佳阈值（0-255 的整数）
       ↓
应用：image.point(lambda p: 255 if p > threshold else 0)
       → 生成纯黑白二值图像
```
